Installation and Environment setup for TypeScript

TS crash course by Arin Bagul

This section has two parts. In first part we will install TypeScript. And in second part we will install extensions to make our like easy while developing our applications using TypeScript.

Part 1: Installation

  1. Install nodejs: Open your web browser and search for "nodejs install" or simply click on given link: https://nodejs.org/en and click on download.
    And simply install it like any other setup just click on next-next-install.
  2. Install VS Code: Open your web browser and search for "vs code download" or simply click on given link: https://code.visualstudio.com/download and click on download.
    And simply install it like any other setup just click on next-next-install.
  3. Open vs code in any folder. By opening a folder or creating a new one, in address bar of windows explorer type cmd and hit enter.
    Command Prompt will appear, now type code . [ code (space) (dot) ] and hit enter. VS code will open in that particular folder.
  4. Open terminal using shortcut ctrl + ` ( control key + backtick key).
  5. In the terminal type, node -v and hit enter, it will display the version of node, it is to ensure that nodejs is properly installed.

    Again type, npm -v and hit enter it will display the version of npm, it is to ensure that npm is properly installed.

    Now type npm i typescript -g. It will install typescript globally in your system.

TypeScript is now successfully installed. Now, we will install the extensions in vs code, extension gives us some extra functionality which will help us in development process.

Part 2: VS code Extensions and environment setup

  1. Use a shortcut ctrl + shift + x. In left side a extension panel appears.
  2. Search for following extensions one by one:
    1. JavaScript and TypeScript Nightly
    2. ESLint
    3. Prettier - Code formatter
    4. Prettier ESLint
    5. Pretty TypeScript Errors
    6. Live Server
  3. Install these extensions by clicking on install button.

Now our TypeScript environment is ready for development process. Lets dive into next chapter: TS file extension.
